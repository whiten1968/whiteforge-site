Image Sanitizer

Every photo from a phone or camera carries hidden data — the GPS coordinates of where you took it, the device that took it, sometimes a thumbnail of the original pre-edit version. Send the photo, share all of that with whoever receives it. Image Sanitizer reads the hidden data, strips it cleanly, and verifies the result — without touching the visible image. Local-only.

Why this exists

In December 2012, the antivirus pioneer John McAfee was on the run from authorities in Belize. A Vice reporter posted a photo of him on the magazine’s website. Within hours, readers had extracted the EXIF metadata embedded in the file and pinpointed his GPS coordinates — a hotel near Rio Dulce, Guatemala. He was arrested by Guatemalan police shortly after.

That was over a decade ago. The hidden data is still there, in every photo from every modern phone and camera, by default. Some social platforms strip some of it on upload (Twitter, Instagram and Facebook remove GPS); many don’t (Discord, email, Slack, file-sharing services). The only way to be sure is to remove it yourself before the file leaves your device.

Image Sanitizer surfaces exactly what’s in the photo you opened, lets you strip it with one click, and verifies the result before reporting success.

How it works

  • Inspect first

    Pick a JPEG and Image Sanitizer reports every metadata segment present: EXIF (camera and capture data, often GPS), XMP (Adobe edit history and ratings), IPTC (captions, byline, copyright), JFIF (producer name), Adobe APP14, freeform comments, and any other application metadata markers.

  • One-click sanitise

    Click Sanitise and Image Sanitizer walks the JPEG’s marker stream, skipping every APPn and COM segment, and writes a fresh JPEG alongside the original. The pixel data (the visible image) is preserved exactly — same dimensions, same compression, byte-identical for the parts that aren’t metadata.

  • Verification on every save

    After every sanitisation, Image Sanitizer re-opens the output and confirms zero APPn and COM markers remain. If anything leaked through, the tainted file is deleted and you see a clear failure — you don’t accidentally ship a file you think is clean.

  • The original is never touched

    The output is always written to a new path of your choosing (suggested name: <original>-sanitized.jpg). The input JPEG remains exactly as you opened it.

v0.1 limitations

Image Sanitizer v0.1 supports JPEG only (.jpg, .jpeg). HEIC (the default on modern iPhones), PNG, TIFF and WebP support is planned for v0.2 onwards.

If you have a HEIC photo you need to sanitise today, the quick workaround is to open the photo on a Mac and use File → Export As… with JPEG format selected. The export drops most of the embedded metadata as a side effect; Image Sanitizer can then verify the result. Native HEIC support is on the way.

What Image Sanitizer does not do

Image Sanitizer does not redact or blur visible content in the image — it only removes invisible metadata. To blur or black out a face, plate, or other visible region you’ll want a dedicated image editor.

Image Sanitizer does not certify legal or regulatory compliance. It helps remove common categories of hidden data; it doesn’t certify your workflow against any specific standard.

Privacy

Image Sanitizer makes no network connections by default. JPEGs are processed entirely on your device. There are no analytics, no telemetry, no servers.

An optional auto-update check can be turned on in Settings → Updates, which makes one HTTPS request per launch to GitHub to look for a new version. No image content, no telemetry, no user data is sent — see the privacy policy for the full detail.

Requirements

macOS 11 (Big Sur) or later. Apple Silicon and Intel both supported — separate disk images for each. Distributed as a Developer ID signed and Apple-notarized disk image — not via the Mac App Store.
